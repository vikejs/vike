Guide to migrate from <Link href="/vike-photon">`vike-photon`</Link> to <Link href="/server">`+server.js`</Link>.

> `vike-photon` is deprecated — it has been replaced by `+server`.
>
> See: <Link href="/blog/universal-deploy" />

> If you're still using `vike-server`, see also: <Link href="/migration/vike-photon" />

## 1. Update dependencies

```shell
npm uninstall vike-photon
npm install vike@latest
```

## 3. Update `+config.ts`

Remove `vike-photon` extension and `+photon` settings.

```ts
// pages/+config.ts

import type { Config } from 'vike/types'
import vikePhoton from 'vike-photon/config' // [!code --]

export default {
  extends: [
    vikePhoton // [!code --]
  ],
  photon: { // [!code --]
    server: 'server/index.ts' // [!code --]
  } // [!code --]
} satisfies Config
```

If you used the `isr` or `edge` settings, they now live at `+vercel.{isr,edge}`:

```ts
// pages/+config.ts

export default {
  // ...
  isr: { expiration: 30 }, // [!code --]
  edge: true, // [!code --]
  vercel: {
    isr: { expiration: 30 }, // [!code ++]
    edge: true // [!code ++]
  }
}
```

## 4. Move server entry to `+server.ts`

Rename your server entry to <Link href="/server">`+server.js`</Link> (we recommend to move it at your root directory), and use the standard syntax `export default { fetch }`.

## 5. `serve()` options

Most `serve()` options have their equivalent, see <Link href="/server#settings" />.

<Warning>
  <p>Previously, `serve()` options applied to both the development and production server.
    Now, those options only apply to the production server.</p>

  If you need the equivalent options for the dev server, use [Vite's server settings](https://vite.dev/config/server-options).

  🚧 Server settings are now separate because values usually differ between dev and prod. If you have a use case where settings share the same value in dev and prod, then [reach out](https://github.com/vikejs/vike/discussions) and we'll create server settings that apply to both dev an prod.
</Warning>


> Note that `createServer` is now built-in and can safely be removed.

> Advanced `serverOptions` should now be passed to `prod.node`, `prod.bun` or `prod.deno`, see [srvx > Runtime Specific Options](https://srvx.h3.dev/guide/options#runtime-specific-options).

## 6. `+photon` options

Remove all `photon` options and replace them with the following.

```ts
// pages/+config.ts

import type { Config } from 'vike/types'

export default {
  photon: { // [!code --]
    // ... // [!code --]
  } // [!code --]
} satisfies Config
```

#### `photon.standalone`

Replace with [standaloner](https://github.com/nitedani/standaloner#readme):

```shell
npm i standaloner
```

```ts
// vite.config.ts

import { defineConfig } from 'vite'
import standaloner from 'standaloner/vite' // [!code ++]

export default defineConfig({
  plugins: [
    standaloner() // [!code ++]
  ]
});
```

#### `photon.compress`

Replace with [compress](https://universal-middleware.dev/middlewares/compress):

```shell
npm i @universal-middleware/compress
```

```ts
// +server.ts

import { Hono } from 'hono'
import vike from '@vikejs/hono'
import compress from '@universal-middleware/compress' // [!code ++]

const app = new Hono()
vike(app) // [!code --]
// Add universal middlewares here (same for Express.js, Fastify, ...) // [!code ++]
vike(app, [compress()]) // [!code ++]
```

#### `photon.static`

Replace with the `prod.static` setting in <Link href="/server">`+server.js`</Link>:

```ts
// +server.ts

export default {
  // ...
  prod: {
    // Serve static files automatically
    static: true, // Default value // [!code ++]
    // Can be disabled:
    static: false, // [!code ++]
    // Or serve from a custom folder:
    static: './public' // [!code ++]
  }
}
```

## 7. Typescript (optional)

If you use TypeScript and `pageContext.runtime`:

```ts
declare global {
  namespace Vike {
    // @docpress-uncomment interface Photon {  // [!code --]
    interface Server { // [!code ++]
      // Pick your server (for correct pageContext.runtime type)
      server: 'hono' // 'express' | 'fastify' | 'hattip' | 'srvx' | 'elysia' | 'h3'
    }
  }
}
```
