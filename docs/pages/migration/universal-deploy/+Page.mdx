Guide to migrate from <Link href="/vike-photon">`vike-photon`</Link> to native `universal-deploy`.

> `vike-photon` is being replaced by a native implementation of universal deployment in Vike.

## 1. Update dependencies

```shell
npm uninstall vike-photon @photonjs/hono @photonjs/express @photonjs/fastify @photonjs/h3 @photonjs/elysia @photonjs/cloudflare @photonjs/vercel
npm install vike@latest
```

You should also install the corresponding `@vikejs/*` package for your server.

## 3. Update `+config.ts`

Remove `vike-photon` extension and move `photon` settings to top-level or `+server.ts`.

```ts
// pages/+config.ts

import type { Config } from 'vike/types'
import vikePhoton from 'vike-photon/config' // [!code --]

export default {
  extends: [
    // ...
    vikePhoton // [!code --]
  ],
  photon: { // [!code --]
    server: 'server/index.ts' // [!code --]
  } // [!code --]
} satisfies Config
```

If you used `isr` or `edge` settings, they are now built-in:

```ts
// pages/+config.ts

export default {
  // ...
  isr: { expiration: 30 }, // [!code ++]
  edge: true // [!code ++]
}
```

## 4. Move server entry to `+server.ts`

Replace `@photonjs/*` with `@vikejs/*`. Your server entry should now be named `+server.ts` at the root of your project, and use the `export default { fetch }` syntax.

## 5. `serve()` options

<Warning>
  <p>Previously, `serve()` options impacted both the dev server and the production server.
    Now, those options only impact the production server.</p>

  If you need the equivalent options for the dev server, please refer to the [Vite documentation](https://vite.dev/config/server-options).
  <p></p>
</Warning>

Most `serve()` options have their equivalent in `+server.ts`, under the `prod` property:

#### `port`

```ts
// +server.ts

export default {
  // ...
  prod: { // [!code ++]
    port: 3000 // [!code ++]
  } // [!code ++]
}
```

#### `hostname`

```ts
// +server.ts

export default {
  // ...
  prod: { // [!code ++]
    hostname: 'localhost' // [!code ++]
  } // [!code ++]
}
```

#### `onReady`

```ts
// +server.ts

export default {
  // ...
  prod: { // [!code ++]
    onReady(server) { // [!code ++]
      // `server` is now an instance of srvx Server. See https://srvx.h3.dev/guide/server
    }, // [!code ++]
  } // [!code ++]
}
```

#### `onCreate`

```ts
// +server.ts

export default {
  // ...
  prod: { // [!code ++]
    onCreate(server) { // [!code ++]
      // `server` is now an instance of srvx Server. See https://srvx.h3.dev/guide/server
    }, // [!code ++]
  } // [!code ++]
}
```

#### `createServer`

This is now built-in. This can safely be removed.

#### TLS options (`serverOptions`)

```ts
// +server.ts

export default {
  // ...
  prod: { // [!code ++]
    tls: { // [!code ++]
      cert: './server.crt', // [!code ++]
      key: './server.key', // [!code ++]
      // optional
      passphrase: 'passphrase' // [!code ++]
    } // [!code ++]
  } // [!code ++]
}
```

#### Advanced `serverOptions`

Those options should now be passed to `prod.node`, `prod.bun` or `prod.deno`. See https://srvx.h3.dev/guide/options#nodejs

## 6. Advanced `photon` options

Make sure that all `photon` options are removed from `+config.ts`:

```ts
// pages/+config.ts

import type { Config } from 'vike/types'

export default {
  photon: { // [!code --]
    // ... // [!code --]
  } // [!code --]
} satisfies Config
```

#### `photon.standalone`

Replace with [standaloner](https://github.com/nitedani/standaloner/?tab=readme-ov-file#with-vite)

```ts
// vite.config.ts

import { defineConfig } from 'vite'
import standaloner from 'standaloner/vite'

export default defineConfig({
  plugins: [standaloner()]
});
```

#### `photon.compress`

Replace with [compress](https://universal-middleware.dev/middlewares/compress)

```ts
// +server.ts

import compress from '@universal-middleware/compress'
import { Hono } from 'hono'
import vike from '@vikejs/hono'

const app = new Hono()
// Add middleware here
vike(app, [compress()]) // [!code ++]
```

#### `photon.static`

Replace with `static` option in your server entry

```ts
// +server.ts

export default {
  // ...
  prod: {
    // Will serve static files automatically (default)
    // @docpress-uncomment static: true // [!code ++]
    // Can be disabled with
    // @docpress-uncomment static: false // [!code ++]
    // Or serve from a custom folder
    // @docpress-uncomment static: './public' // [!code ++]
  }
}
```
