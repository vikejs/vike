[Deno](https://deno.land) is a runtime for JavaScript and TypeScript that uses [V8](https://v8.dev/) and is built in Rust. It supports zero config typescript and has broad npm compatibility.

It also supports vike.

## `package.json` and `vite.config.ts`

If you are porting from a Node.js project, you likely already have a package.json file. If you create a new project we suggest to create a minimal one like this:
 
package.json:
```json
{
  "dependencies": {
    "vite": "latest",
    "vike": "latest"
  }
}
```

> In a new Deno project do not run any package manager â€” let Deno handle the dependencies.

### Vite config without package.json

If you don't want to use a `package.json` file, you can use the `npm:` specifier.

```ts
// vite.config.ts
import { defineConfig } from 'npm:vite'
import vike from 'npm:vike@latest/plugin'
 
export default defineConfig({
  plugins: [vike()]
})
```

#### Node based Deno projects

If followed the [approach earlier](#package-json-and-vite-config-ts) and imported node packages via the `package.json` you can use the linked libraries from there throughout the project without the `npm:`-prefix.

```ts
// vite.config.js
import { defineConfig } from 'vite' // note the missing `npm:` prefix
import vike from 'vike/plugin'

export default defineConfig({
  plugins: [vike()]
})
```

> In any case a vite config file is required.

## Deno configuration

We let deno handle our dependencies, so we need to add the `--node-modules-dir` flag to deno commands.

example `deno.json` in project root:
```json
{
  "tasks": {
    "dev": "deno run --allow-all --node-modules-dir npm:vike dev",
    "build": "deno run --allow-all --node-modules-dir npm:vike build",
    "preview": "deno run --allow-all --node-modules-dir npm:vike preview",
    "prod": "deno task build && deno task preview"
  },
}
```

Now you can run the following commands:
 - `deno task dev` to start a development server
 - `deno task build` to build the project
 - `deno task preview` to preview a production build
 - `deno task prod` to build and preview the project

More on how Deno handles dependencies: https://docs.deno.com/runtime/fundamentals/modules/#importing-third-party-modules-and-libraries
