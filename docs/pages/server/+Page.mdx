The `+server.ts` (or `+server.js`) file defines your server entry point. It's the recommended way to integrate Vike with your server and to enable universal deployment.

It should export a default object with a `fetch` property. This makes your server compatible with various platforms and allows Vike to automatically deploy it.

```ts
// +server.ts

import { Hono } from 'hono'
import vike from '@vikejs/hono'

const app = new Hono()
vike(app)

export default {
  fetch: app.fetch
}
```

## Integrating Vike with your server

Vike provides a middleware that can be used with any server framework. It handles the rendering of your pages.

## Universal Deployment

When you define a `+server.ts` file, Vike can automatically build and deploy your app to various platforms like <Link href="/vercel">Vercel</Link> or <Link href="/cloudflare">Cloudflare</Link>.

## Supported options

### `static`

```ts
// +server.ts

export default {
  // ...
  // Will serve static files automatically (default)
  // @docpress-uncomment static: true // [!code ++]
  // Can be disabled with
  // @docpress-uncomment static: false // [!code ++]
  // Or serve from a custom folder
  // @docpress-uncomment static: './public' // [!code ++]
}
```
