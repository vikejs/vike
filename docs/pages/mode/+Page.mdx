Vike's `+mode` setting is just an alias for [Vite's `mode` setting](https://vite.dev/guide/env-and-mode.html#modes).

You can set it over Vike's <Link href="/cli">CLI</Link>:

```bash
$ vike build --mode staging
```

Or over the `VIKE_CONFIG` environment variable:

```bash
$ VIKE_CONFIG="{mode:'staging'}" vike build
```

Both are equivalent to setting Vite's `mode` setting:

```ts
// vite.config.ts

import type { UserConfig } from 'vite'

export default {
  mode: 'staging'
} satisfies UserConfig
```

> Unlike most Vike settings, `+mode` can't be defined inside a `+config.js` file (Vike emits a warning if you try): it's resolved before Vike loads your `+config.js` files. Set it over the CLI, the `VIKE_CONFIG` environment variable, `vite.config.js`, or <Link href="/api">Vike's API</Link> instead.
